## Task 2: 跨 Segment 文本去重 ✅ 已完成

**实际实现**：集成到 `TextProcessor`（v4 架构已有），而非新建独立文件

### 改动文件

1. **`text_processor.py`** — 新增属性和方法
   - `_prev_segment_tail`、`_TAIL_WINDOW=20`、`_HEAD_WINDOW=20`、`_MIN_OVERLAP=2`
   - `dedup_overlap(text)` 方法（第 178-230 行）
   - `reload()` 和 `reset()` 同步清理 `_prev_segment_tail`

2. **`model_server.py`** — 在 `_punc_worker()` 中集成（第 520 行）
   ```python
   cleaned = self._text_processor.dedup_overlap(cleaned)
   ```
   调用位置：`preprocess()` 之后，`tick()` 之前

### 算法

```
prev_tail = "翻面。去皮"[-20:]  → "。去皮"
new_head  = "皮皮叮叮椒王..."[:20] → "皮皮叮叮椒王一根，..."
重叠查找: tail后缀 vs head前缀的最长匹配
  - "皮" (len=1, < MIN_OVERLAP) → 不处理
  - 无更多重叠

prev_tail = "去皮皮"[-20:]  → "去皮皮"
new_head  = "皮皮叮叮椒王..."[:20]
重叠查找:
  - "皮皮" (len=2, ≥ MIN_OVERLAP) → 切掉 "皮皮"
  → 输出 "叮叮椒王..."
```

### 为什么放在 TextProcessor 而非独立文件

v4 架构中 `TextProcessor` 已有：
- 精确匹配去重（`_is_duplicate` / `_recent_outputs`）
- 预处理管线（`preprocess` → `tick`）
- 截断处理（`append_truncated`、`_header` 机制）

新增的跨 segment 重叠去重天然属于文本后处理范畴，放在 TextProcessor 保持架构内聚。
